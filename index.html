The Solution: Zero-Trust Workflow

Scan. Upload. Print. Vanish.

1. Scan QR Code

Customer scans a unique QR code at the cybercafe terminal.

2. Direct Upload

Document is uploaded via mobile web to a secure cloud bucket.

3. Encrypted View

Owner sees only Filename and Page Count. The content is hidden.

4. Vanish

File is auto-deleted immediately after the print job completes.

Technology Stack

Next.js Web App for mobile uploads without installation.

Backend

Node.js & Socket.io for real-time bi-directional communication.

Desktop Agent

Flutter app running on the cafe terminal to handle local printer drivers.

Encryption

AES-256 Client-Side Encryption ensuring data is unreadable in transit.

Zero-Trust

The underlying security philosophy driving the entire architecture.

Architecture & Security Pillars

Data Flow Overview

  1. **Client (Mobile Web App):** Encrypts the file using AES-256 before upload.
  2. **Cloud (AWS S3):** Stores the encrypted blob temporarily. Lifecycle policy set to 'Delete after 5 mins'.
  3. **Socket Server:** Signals the Flutter App on the terminal that a "New Job is Ready".
  4. **Terminal (Electron App):** Downloads the encrypted blob, decrypts **in memory**, and pipes the stream directly to the local printer driver.

No files ever hit the terminal's disk.

Auto-Destruct (Hot Data)

A rigid TTL (Time-To-Live) policy ensures no file remains on the server for more than **5 minutes**. Maximize ephemerality.

Blind Processing

The cybercafe owner never possesses the file. The print process is automated, bypassing the local "Downloads" folder entirely.
